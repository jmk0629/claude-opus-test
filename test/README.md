# Claude Code 하네스 엔지니어링 예제 (튜토리얼)

> 초보자도 이해할 수 있는 에이전트 & 서브에이전트 패턴 가이드

> ℹ️ 이 디렉토리(`test/`)는 **학습/연습용 튜토리얼**입니다.
> 실전용 에이전트/커맨드(예: `/ingest-backend`)는 상위 디렉토리(`../agents/`, `../commands/`)에 있습니다.

---

## 이게 뭔가요?

Claude Code에서는 **나만의 전문가 팀**을 만들 수 있습니다.

예를 들어:
- `기획자 에이전트` → "15년차 기획자처럼 요구사항을 분석해"
- `개발자 에이전트` → "시니어 개발자처럼 코드를 짜"
- `테스터 에이전트` → "QA 전문가처럼 버그를 찾아"
- `리뷰어 에이전트` → "코드 리뷰어처럼 품질을 검사해"

이 전문가들이 **서로 협력**해서 하나의 작업을 완수합니다.

---

## 왜 이 패턴을 쓰나요? (토큰 절약의 비밀)

```
❌ 나쁜 방법 (토큰 낭비):
메인 Claude가 혼자 모든 파일을 읽고, 기획하고, 코딩하고, 테스트까지 = 컨텍스트가 엄청나게 커짐

✅ 좋은 방법 (하네스 패턴):
메인 Claude는 "지휘관" 역할만
서브에이전트들이 각자 맡은 부분만 집중해서 처리 = 각자 작은 컨텍스트로 처리
```

**핵심 원리**: 서브에이전트는 **완전히 새로운 빈 컨텍스트**에서 시작합니다.
메인 대화의 긴 히스토리를 들고 다니지 않아요. 그래서 토큰이 절약됩니다.

```
메인 컨텍스트: [긴 대화 히스토리... 1만 토큰]
    ↓ 서브에이전트 A 실행
서브에이전트 A: [작업 설명만... 500토큰] → 결과 반환 → 메모리 해제
    ↓ 서브에이전트 B 실행  
서브에이전트 B: [작업 설명만... 500토큰] → 결과 반환 → 메모리 해제
```

## 디렉토리 구조

```
harness-engineering-test/          ← 루트 (실전 세트)
├── agents/                        ← 실전 서브에이전트 (controller-/service-/... -analyzer)
├── commands/                      ← 실전 커맨드 (/ingest-backend)
├── memory/                        ← 프로젝트 메모리 (심볼릭 링크)
└── test/                          ← 👈 여기 (튜토리얼)
    ├── README.md                  ← 지금 읽고 있는 파일
    ├── CONCEPTS.md                ← 핵심 개념 심층 이해
    │
    ├── agents/                    ← 튜토리얼 에이전트
    │   ├── planner.md             ← 기획자 에이전트
    │   ├── developer.md           ← 개발자 에이전트
    │   ├── tester.md              ← 테스터 에이전트
    │   └── reviewer.md            ← 코드 리뷰어 에이전트
    │
    ├── commands/                  ← 튜토리얼 커맨드
    │   ├── build-feature.md       ← /build-feature (전체 워크플로우)
    │   └── quick-review.md        ← /quick-review (빠른 리뷰)
    │
    └── examples/                  ← 실제 사용 예시
        └── token-savings-demo.md  ← 토큰 절약 원리 데모
```

`CONCEPTS.md` 파일을 읽으세요.
