# 빠른 코드 리뷰

`$ARGUMENTS`에 대해 reviewer와 tester 에이전트를 **동시에** 실행해서 빠르게 품질을 확인합니다.

---

## 병렬 검토 실행

다음 두 에이전트를 **동시에** 실행하세요 (순서 없이 병렬):

**reviewer 에이전트**:
- 작업: "다음 코드를 리뷰해줘: $ARGUMENTS (없으면 git diff). 신뢰도 75점 이상 이슈만 보고해. 보안 취약점과 명백한 버그에 집중해줘."

**tester 에이전트**:
- 작업: "다음 코드를 분석해서 누락된 테스트 케이스와 잠재적 버그를 찾아줘: $ARGUMENTS (없으면 git diff). 가장 위험한 엣지 케이스 3가지를 최우선으로 보고해줘."

---

## 결과 통합

두 에이전트 완료 후:

1. 겹치는 이슈를 합쳐서 중복 제거
2. 심각도 순으로 정렬
3. 사용자에게 통합 보고서 제시:

```
## 빠른 리뷰 결과

### 즉시 수정 필요
[reviewer + tester 모두 지적한 이슈, 또는 심각도 높은 이슈]

### 권장 수정
[reviewer 또는 tester 중 하나가 지적한 이슈]

### 결론
[머지 가능 여부]
```

---

## 사용 예시

```bash
# 특정 파일 리뷰
/quick-review src/auth/login.ts

# git diff 기준 리뷰 (인자 없이)
/quick-review

# 여러 파일
/quick-review src/api/users.ts src/models/User.ts
```

**왜 병렬로 실행하나요?**
reviewer는 "코드 품질" 렌즈로, tester는 "버그 가능성" 렌즈로 봅니다.
같은 코드를 다른 관점에서 동시에 검토하면 더 많은 문제를 빠르게 발견할 수 있습니다.
순차 실행하면 2배 시간이 걸리지만, 병렬 실행하면 1배 시간에 2배 관점을 얻습니다.
