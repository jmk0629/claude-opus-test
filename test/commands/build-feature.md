# 기능 개발 워크플로우

전문가 에이전트 팀을 활용해서 `$ARGUMENTS` 기능을 체계적으로 개발합니다.

---

## Phase 1: 기획 분석

**목표**: 무엇을 만들어야 하는지 명확히 정의

TodoWrite로 전체 진행상황을 추적하는 할 일 목록을 만드세요:
- [ ] Phase 1: 기획 분석
- [ ] Phase 2: 코드베이스 탐색
- [ ] Phase 3: 구현
- [ ] Phase 4: 테스트
- [ ] Phase 5: 코드 리뷰
- [ ] Phase 6: 완료 보고

그런 다음 **planner 에이전트**를 실행하세요:
- 작업: "$ARGUMENTS 기능의 요구사항을 분석하고 사용자 스토리와 엣지 케이스를 포함한 기획서를 작성해줘. 프로젝트에 관련 파일이 있다면 읽어서 현재 상태도 파악해줘."
- 에이전트가 완료되면 기획서를 사용자에게 보여주세요
- **사용자 확인을 기다리세요** ("이 방향으로 진행할까요?")

---

## Phase 2: 코드베이스 탐색

**목표**: 기존 코드 패턴 파악 (구현 전 필수)

사용자 승인 후, **code-explorer 에이전트 2개를 병렬로 실행**하세요:

에이전트 A:
- 작업: "$ARGUMENTS 기능과 유사한 기존 구현을 찾아서 파일 경로, 패턴, 코드 스타일을 분석해줘. 구현에 꼭 읽어야 할 핵심 파일 5개 목록도 알려줘."

에이전트 B:
- 작업: "$ARGUMENTS 기능이 영향을 미칠 모듈의 아키텍처를 분석해줘. 컴포넌트 관계, 데이터 흐름, 의존성을 파악해줘."

두 에이전트가 완료되면:
1. 핵심 파일들을 직접 읽어서 내용을 파악하세요
2. 발견된 패턴과 컨벤션을 요약하세요

---

## Phase 3: 구현

**목표**: 기획서와 코드베이스 패턴에 맞는 구현

**구현 시작 전 사용자에게 반드시 확인하세요**:
- 기획서에서 불명확한 부분
- 구현 방향 (여러 방법이 있다면 옵션 제시)
- 예상 작업 범위

사용자 승인 후 **developer 에이전트**를 실행하세요:
- 작업: "아래 기획서와 코드 패턴을 기반으로 $ARGUMENTS 기능을 구현해줘.
  
  [기획서 내용 붙여넣기]
  
  발견된 패턴: [Phase 2에서 발견한 패턴 요약]
  
  반드시 기존 코드 스타일과 패턴을 따르고, 구현 완료 후 변경된 파일 목록을 보고해줘."

---

## Phase 4: 테스트

**목표**: 구현된 기능의 버그와 누락 케이스 발견

**tester 에이전트**를 실행하세요:
- 작업: "$ARGUMENTS 기능 구현이 완료됐어. 아래 파일들을 테스트해줘:
  
  [developer 에이전트가 생성/수정한 파일 목록]
  
  기획서의 엣지 케이스: [Phase 1의 엣지 케이스 목록]
  
  버그 발견 시 재현 방법과 수정 제안을 포함해서 리포트해줘."

테스터가 심각한 버그를 발견했다면:
- 사용자에게 보고하고
- developer 에이전트로 수정 후 재테스트

---

## Phase 5: 코드 리뷰

**목표**: 머지 전 최종 품질 검증

**reviewer 에이전트**를 실행하세요:
- 작업: "다음 파일들의 코드 리뷰를 해줘:
  
  [구현된 파일 목록]
  
  신뢰도 75점 이상의 이슈만 보고해줘. 보안, 정확성, 유지보수성 위주로 검토해줘."

리뷰 결과를 사용자에게 보여주고:
- 반드시 수정할 이슈가 있으면 developer 에이전트로 수정
- 선택적 이슈는 사용자가 결정

---

## Phase 6: 완료 보고

TodoWrite로 모든 항목을 완료 처리하고, 다음을 요약해서 보고하세요:

```
## 기능 개발 완료: [기능명]

### 구현 내용
- 무엇을 만들었나

### 변경된 파일
- 파일 목록 (파일경로:라인번호)

### 주요 결정사항
- 결정 1과 이유
- 결정 2와 이유

### 권장 후속 작업
- 다음에 할 수 있는 것들
```
